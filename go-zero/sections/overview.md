# 概览 / 适用场景
- 名称：`@mfycommon/go-zero` v1.8.2，提供基于 go-zero 的 REST/RPC/任务服务规范、治理与示例。
- 推荐用在：中小型 API/网关/内部 RPC，追求脚手架+治理一体化（限流、熔断、超时、日志、trace）。
- 替代方案：核心领域/高度复杂规则用 `@mfycommon/DDD`；极简 API 用标准库+轻量路由；高并发长连接/自定义 pipeline 用自研。
- 不适用：需要细粒度中间件管线且 go-zero 难以满足的场景；领域不变量强依赖聚合的核心域。
- 何时使用：需快速交付 REST/RPC，统一日志/trace/metrics，复用 goctl 生成与内置治理；避免自建限流/熔断框架。***
