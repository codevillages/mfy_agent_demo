# 概览 / 适用场景
- 名称：`@mfycommon/gorm` v1.22.4，面向 MySQL 的 ORM 使用规范（含迁移、查询、事务、性能与可观测）。
- 推荐用在：中小型业务表的常规 CRUD/查询/事务操作，需要链路追踪、连接池管理、模型标签统一。
- 替代方案：纯 SQL 用 `database/sql` + `sqlx`；高性能极致场景用原生 SQL + 手写扫描；简单 KV/缓存类不必用 ORM。
- 不适用：极端性能要求、复杂跨表链路需要精确 SQL 优化且 ORM 抽象有损；需要多数据库特性的场景（仅覆盖 MySQL）。
- 何时使用：需要快速建表/生成模型，自动迁移/Hook，统一软删除/时间字段/审计字段，对接我们的日志、trace、metrics 封装。***
