# Cluster / PubSub 特殊错误

## Cluster
- `MOVED` / `ASK`：客户端自动重定向，无需人工处理；持续出现需检查集群扩缩容是否完成。
- `CLUSTERDOWN`：暂时不可用，可按网络错误策略重试；持续失败要切流或熔断。
- `READONLY`：主从切换时短暂可见，通常可重试；若频繁出现检查路由和权限。

## PubSub
- 订阅失败：`Subscribe` 返回错误直接失败；应在启动阶段校验。
- 通道关闭：`Channel()` 读到关闭需退出消费；根据需要重建订阅。
- 订阅阻塞：长时间无消息会阻塞读取，需结合 `context` 或超时避免泄漏。
